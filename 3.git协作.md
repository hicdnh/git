#### 远程分支更新到本地
##### 本地没有冲突代码
场景：早晨上班，其他人昨天晚上已提交所有代码，并且没有和你有编辑相同模块的；
这时候，更新内容不会有冲突；
命令：
```
git fetch origin main
# 把远程的main分支更新到本地的main分支
```

##### 和本地代码有冲突：已经git add，没有git commit
场景：
* 同一个模块，本地已经修改
* 已经git add
处理方法：
* 无需处理
